Use Reset to setup your components

Posted on 2020-01-19 | In tips

The Reset Unity message can be used to setup SerializeField fields of your prefabs.

Problem

Many times I see something like that:

(Awake is most common, but I also seen someone doing that in Start).

This is valid and works, but not as good as add some extra work and assign _label field in the Inspector, so you can avoid the Awake and GetComponent call on spawning GameObject. But no one likes drag’n’dropping too much, right?

Solution

Simply change the Awake() to Reset().

Reset is invoked by Unity Editor everytime your MonoBehaviour is assigned to GameObject, or can be invoked manualy by choose Reset from component context menu. So when you add your MyLabel component it will automatically assign Text to correct field, so you don’t have do the drag and drop thing.

Also, by using Reset you can use some of the heavier methods (like FindComponentInHierarchy), because it’s not running in gameplay time.
